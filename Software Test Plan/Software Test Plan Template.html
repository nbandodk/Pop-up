1. Overview

This section provides an overview of the testing approach used to verify the software product.

1.1 Test Objectives

This test plan describes the testing approach and overall framework that will drive the testing of Pop-up's initial setup(databases and connectivity to database) and also the first core feature.
The primary objective will be to make sure there are no errors and faulty functionality in the following implemented core feature: Login/Registration.
Through these carefully curated test cases we intend to remove and rectify any discrepancies that might have crept into the feature of this sprint.

1.2 Test Environment

Pop-up's Login/Registration page will be hosted locally as well as at SoIC web servers.
The database for the same will also be hosted locally and on db.soic.indiana.edu.
This test phase will be carried out on standard laptops available in the market right now from OEMs such as Dell, Lenovo, Asus, Apple, etc. We believe that the test environment will be similar to the oprational environment for the tests stated in this document.
No additional support or software packages will be required for these tests. All computers must have an active internet connection at the time of testing.

A Windows or Mac environment with Internet explorer 8, 9 and 10+, Firefox 27.0+, as well as Google Chrome 32.0+ should be available to each tester.

The testing environment will also need the following for local hosting of the website: XAMPP.

1.3 Test Personnel

The personnel involved in this testing phase will initially only include our team members.
Once we have finished all the exhaustive tests from our side, we will again do testing along with the customers.
As team members we will first role play as the end user and carry out the tests.
During the testing with the customer, the customer will be the end user and will try out all the possible actions in the first core feature, i.e. Login/Registration.
The roles of our team members will be as follows: Develop test conditions, test cases, expected results, and execution scripts. Perform execution and validation. Re-test after software modifications have been made. Prepare testing metrics and provide regular status.

1.4 Acceptance Criteria

The Acceptance Criteria for this testing phase will be as follows:
  • The connection between the website and database is successful without any errors, both on test as well as operational environment(test server as well as actual hosted server)
  • User is able to register successfully and all his details are stored in the database.
  • User is able to login with the correct credentials and upon successful login, is redirected to index page.
We will tolerate a maximum of 3 errros before the testing would be abandoned to rectify the errors.

1.5 Noted Omissions

The testing of the following software specification can be omitted:
  • Well defined error messages given in an external pop-up box
This can be ignored since styling will be carried out later with Bootstrap and the echoed error messages are enough to tell the user if any errors are raised.
